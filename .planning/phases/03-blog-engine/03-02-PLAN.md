# Plan 03-02: Blog Pages & Navigation

## Goal
Create blog index with pagination, individual post pages, tag pages, related posts component, and wire the landing page blog teaser to real content collection data.

## Tasks

### Task 1: Blog Post Page (`/blog/[slug]`)
<task>
Create `src/pages/blog/[slug].astro`:
- Use `getStaticPaths()` with `getCollection('posts')` filtering out drafts
- Use `render()` from `astro:content` to get Content component and headings
- Calculate reading time using the utility
- Pass all props to BlogLayout
- Map `post.id` to params.slug
</task>
<verify>
- `npm run build` generates individual post pages
- Build output shows `/blog/building-with-astro` and `/blog/automation-at-scale`
</verify>

### Task 2: Blog Index Page (`/blog`)
<task>
Create `src/pages/blog/[...page].astro`:
- Use `getStaticPaths()` with `paginate()` for pagination
- Query `getCollection('posts')` filtering out drafts, sorted by date descending
- Each entry: title, date (formatted), description, tags, reading time
- Pagination controls at bottom (prev/next) when applicable
- Page size: 10 posts
- Design: clean list layout matching the design system
</task>
<verify>
- `npm run build` generates `/blog/index.html`
- Pagination works (would generate `/blog/2/index.html` etc. with enough posts)
</verify>

### Task 3: Tag Pages (`/blog/tags/[tag]`)
<task>
Create `src/pages/blog/tags/[tag].astro`:
- Use `getStaticPaths()` collecting all unique tags from non-draft posts
- Filter posts by tag, sorted by date descending
- Reuse the same post list design from blog index
- Show tag name as heading with post count
- Link back to `/blog`
</task>
<verify>
- `npm run build` generates tag pages (e.g., `/blog/tags/astro/index.html`)
</verify>

### Task 4: Related Posts Component
<task>
Create `src/components/RelatedPosts.astro`:
- Accept props: currentSlug, tags, allPosts (or query internally)
- Find posts sharing tags with current post, excluding current post
- Score by number of shared tags, take top 3
- Display as compact list: title, date
- Show nothing if no related posts found
</task>
<verify>
- Component renders related posts correctly
- Excludes the current post from results
</verify>

### Task 5: Add Related Posts to BlogLayout
<task>
Update `src/layouts/BlogLayout.astro`:
- Import and render RelatedPosts component before the "Back to blog" link
- Pass current slug and tags
</task>
<verify>
- Related posts appear at bottom of blog post pages
</verify>

### Task 6: Wire Landing Page Blog Teaser
<task>
Update `src/pages/index.astro`:
- Replace hardcoded `blogPosts` array with `getCollection('posts')` query
- Filter out drafts, sort by date descending, take first 3
- Calculate reading time for display
- Link each post to `/blog/[slug]`
- Make the blog teaser entries clickable
</task>
<verify>
- Landing page blog teaser shows real post data
- Links navigate to actual blog post pages
</verify>

## Verification
<verification>
1. `npm run build` completes without errors
2. All blog pages generated (index, individual posts, tag pages)
3. Navigation works: index -> post -> back, index -> tag -> post
4. Landing page blog teaser pulls from real content collection
5. Related posts appear on post pages
6. Reading time displayed correctly on all views
</verification>
