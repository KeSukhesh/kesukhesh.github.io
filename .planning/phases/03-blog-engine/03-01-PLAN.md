# Plan 03-01: Content Infrastructure

## Goal
Set up Astro 6 content collections for blog posts, configure Shiki dual-theme syntax highlighting, create BlogLayout with prose styling, add sample posts, and build a reading time utility.

## Tasks

### Task 1: Content Collection Config
<task>
Create `src/content.config.ts` defining the `posts` collection:
- Import `defineCollection` from `astro:content`
- Import `z` from `astro/zod`
- Import `glob` from `astro/loaders`
- Define schema: title (string), date (date), description (string), tags (array of strings, default []), draft (boolean, default false)
- Use glob loader pointed at `src/content/posts`
- Export as `collections` object
</task>
<verify>
- `npm run build` succeeds with content collection configured
- TypeScript types generated for the collection
</verify>

### Task 2: Sample Blog Posts

Create 2 sample markdown posts in `src/content/posts/`:

1. `building-with-astro.md` — about building this site with Astro
   - title: "Building with Astro: A Modern Static Site"
   - date: 2026-04-01
   - description: "How I built this site with Astro, Tailwind CSS, and a focus on performance."

- `npm run build` processes them without errors
</verify>

### Task 3: Shiki Dual-Theme Config
<task>
Update `astro.config.mjs` to configure Shiki with dual themes:
- Add `markdown.shikiConfig` with light/dark theme pair
- Use `themes: { light: 'github-light', dark: 'github-dark' }` with `defaultColor: false`
- This outputs CSS variables that respect our theme toggle
</task>
<verify>
- Build succeeds
- Code blocks in sample posts render with Shiki classes
</verify>

### Task 4: Reading Time Utility
<task>
Create `src/utils/reading-time.ts`:
- Export function `getReadingTime(content: string): string`
- Calculate word count, divide by 200 (average reading speed)
- Return formatted string like "3 min read"
</task>
<verify>
- Function returns correct reading time for sample content
</verify>

### Task 5: Prose Styling
<task>
Add prose/markdown body styles to `src/styles/global.css`:
- `.prose` class with typography rules matching the design system
- Headings: Libre Baskerville, proper size scale
- Paragraphs: 1.8 line-height, comfortable reading width
- Code blocks: Space Mono, rounded corners, background contrast
- Inline code: subtle background, rounded
- Links: forest green, underline on hover
- Lists: proper indentation and markers
- Blockquotes: left border forest green, italic
- Images: max-width 100%, rounded
- Tables: bordered, striped rows
- Dark mode variants for all
- Shiki dual-theme CSS variables (`[data-theme]` selectors for light/dark)
</task>
<verify>
- Styles render correctly for markdown content
- Dark mode styles apply correctly
</verify>

### Task 6: BlogLayout Component
<task>
Create `src/layouts/BlogLayout.astro`:
- Accept props: title, description, date, tags, readingTime, headings (for ToC)
- Extend BaseLayout (wrap with it)
- Post header: title (h1), date formatted, reading time, tag pills
- Article body with `prose` class wrapping the slot
- Placeholder for related posts (will be added in Plan 03-02)
- Responsive: content max-width ~65ch for readability
- No scroll-snap (blog pages don't use snap-page)
</task>
<verify>
- Component renders without errors
- Layout matches design system (ink, paper, forest colors, Libre Baskerville)
</verify>
